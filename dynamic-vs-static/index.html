Dynamic vs. Static

Rafael Ordog, a.k.a. DeVill

About the language war

It started in the 1970s between Smalltalk (dynamic) and C (static)

Flared up again in the 1980s Smalltalk vs C++

And it never really stopped ever since...

About me

I picked up programing as a hobby in 1988, I used QBasic (dynamically typed)

The first languages I used in production were ActionScript and C++ (statically typed)

Over the years switched languages pretty regularly

Now I use PHP (optionally typed) and Ruby (dynamically typed)

So by now, I should know...

... or at least someone should know!

I think it's the wrong question...

The reason the war does not end, is that they have different advantages.

... and the right question is:

Is it possible to combine them?

Probably not!

(but it's still worth trying...)

... or maybe the right question is:

When to use which?

Let's see some
pros and cons

Static types are safer!

It's possible to catch errors at compile time

On the other hand tests catch more errors!

Static types are annoying!

Oh, how I hated typedef in C++, and templates, and...

But if it's not Java, then by now it has a pretty good type inference

Monkey patching feels really bad in static languages

... and also in dynamic languages

BUT, it makes it really easy to extend the languages through libraries

And it's not just monkey patching...

Dynamic function calls

Calling a function in the context of another object

Infinitely segregated interfaces (no need to specify them)

With dynamic typing there is no way to tell the object type!

FALSE

Run the damn test, will ya!

So what's the verdict?

Verbose code, less chance for hacks in return for more safety

Very concise code, with lot less duplication, but it's easy to make a mess

Which one is better?

Depends on your company

If it's a small start up with UI heavy code, use a dynamic language

If it's a large established enterprise, maybe you need static types

But wait...

... I want my start up to become a large enterprise

Sure! And at some point, you'll start writing your back end services in static languages

And at the end of the day, it all comes down to taste and getting used to

Once you got used to one, it's really hard to work with the other

It doesn't really matter which one is better... maybe neither is
