What are the trade offs of SQL?

It was designed with single machine in mind

Shading is a solution

ACID guaranties become hard to keep up when there are concurrent writes

The the schema is hard to change

The only good way to extend a table is by copying it

Gets even worse when data is denormalized for performance

Complex data is disassembled to several tables

Due to the rigid SQL schema this data is spread on the disk

Pseudo key-value store: JSON in SQL

So if we give up on certain advantages, why not gain something in return?

And that's how NoSQL was born
